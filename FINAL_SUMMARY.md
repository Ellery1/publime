## 问题诊断过程

### 文件对话框问题的调试
通过添加详细的日志输出，我们发现了根本原因：

**问题**: 点击"文件 - 打开"菜单时，`open_file` 方法接收到的参数是 `False` 而不是 `None`

**原因**: Qt 的 `QAction.triggered` 信号会自动传递一个 `checked` 参数（布尔值），当菜单项不可勾选时，这个值是 `False`

**日志输出显示**:
```
[DEBUG] open_file 被调用, file_path=False, type=<class 'bool'>
```

**解决方案**: 修改参数检查逻辑，同时处理 `None`、`False` 和空字符串

## 技术要点

### Qt 信号槽的参数传递
- `QAction.triggered` 信号会传递 `checked` 参数
- 对于不可勾选的菜单项，这个值是 `False`
- 需要在槽函数中正确处理这个参数

### 拖放事件处理
- QPlainTextEdit 默认接受文本拖放
- 需要禁用 `acceptDrops` 让父窗口处理文件拖放

### 主题应用
- 使用 QSS (Qt Style Sheets) 应用主题
- 需要同时设置主窗口和所有子组件

## 开发者注意事项

### Qt 信号槽最佳实践
```python
# 错误的方式
def my_slot(self, file_path: str = None):
    if file_path is None:  # 这会失败，因为可能收到 False
        ...

# 正确的方式
def my_slot(self, file_path=None):
    if file_path is None or file_path is False or file_path == "":
        ...
```

### 文件对话框最佳实践
```python
# 使用 DontUseNativeDialog 确保对话框可见
file_path, _ = QFileDialog.getOpenFileName(
    self,
    "标题",
    "",
    "过滤器",
    options=QFileDialog.Option.DontUseNativeDialog
)
```
