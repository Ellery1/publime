# Publime 重新打包指南

## 问题说明

当前问题：格式化后的SQL在EXE中显示为一行，没有换行符。

**根本原因**：EXE可能使用了缓存的旧代码，需要完全清理并重新打包。

## 验证代码正确性

首先验证代码本身是否正确：

```bash
python verify_sql_format.py
```

如果看到 "✅ 所有测试通过！" 并且显示有22个换行符，说明代码是正确的。

## 重新打包步骤

### 步骤 1: 关闭所有 Publime 进程

**重要！** 必须先关闭所有正在运行的 Publime.exe

1. 关闭所有 Publime 窗口
2. 打开任务管理器 (Ctrl+Shift+Esc)
3. 在"进程"标签中查找 "Publime.exe"
4. 如果找到，右键点击 -> "结束任务"
5. 确保没有任何 Publime.exe 进程在运行

### 步骤 2: 运行清理和打包脚本

```bash
python clean_and_build.py
```

这个脚本会自动：
1. 删除所有 Python 缓存文件 (`__pycache__`, `*.pyc`)
2. 删除 PyInstaller 构建目录 (`build/`, `dist/`)
3. 重新打包 EXE

**如果遇到 "拒绝访问" 错误**：
- 说明 Publime.exe 仍在运行
- 返回步骤 1，确保完全关闭所有进程

### 步骤 3: 测试新的 EXE

1. 运行新的 `dist\Publime.exe`
2. 创建新文件 (Ctrl+N)
3. 粘贴测试SQL（见下方）
4. 按 `Ctrl+Alt+F` 格式化
5. 检查结果

## 验证要点

✓ 关键字小写: `select`, `from`, `where`, `and`, `union all`  
✓ JSON_TABLE 在一行上，格式紧凑  
✓ FROM 子句的表和 JSON_TABLE 用逗号分隔在同一行  
✓ WHERE 条件每个 AND 一行  
✓ UNION ALL 保留  
✓ 名字保留: '王羽', '孙伟东'  
✓ **有多行显示，不是全部在一行**

## 如果问题仍然存在

### 方法 1: 手动清理

如果自动脚本失败，手动执行以下步骤：

1. **关闭所有 Publime.exe 进程**

2. **删除缓存目录**：
   ```bash
   rmdir /s /q build
   rmdir /s /q dist
   rmdir /s /q core\__pycache__
   rmdir /s /q ui\__pycache__
   rmdir /s /q utils\__pycache__
   rmdir /s /q themes\__pycache__
   del Publime.spec
   ```

3. **重新打包**：
   ```bash
   python build_exe.py
   ```

### 方法 2: 重命名 EXE

如果Windows缓存了旧的EXE：

1. 打开 `build_exe.py`
2. 修改第 20 行：
   ```python
   '--name=Publime',  # 改为 '--name=Publime_v2',
   ```
3. 重新运行：
   ```bash
   python build_exe.py
   ```
4. 新的EXE将是 `dist\Publime_v2.exe`

### 方法 3: 清理 PyInstaller 缓存

```bash
# 清理 PyInstaller 临时文件
rmdir /s /q %TEMP%\_MEI*
```
